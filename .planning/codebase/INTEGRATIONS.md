# External Integrations

**Analysis Date:** 2026-03-03

## APIs & External Services

**Email & Transactional Services:**
- Resend - Transactional email delivery for invites, reminders, session summaries, password resets
  - SDK/Client: `resend` npm package
  - Auth: `RESEND_API_KEY` environment variable
  - Email templates implemented via React Email in `src/lib/email/templates/`

**OAuth & Identity Providers:**
- Google OAuth 2.0 - "Sign in with Google" for Google Workspace integration
  - Auth: `AUTH_GOOGLE_ID`, `AUTH_GOOGLE_SECRET` environment variables
- Microsoft OAuth 2.0 - "Sign in with Microsoft" for Azure AD / Entra ID integration
  - Auth: `AUTH_MICROSOFT_ID`, `AUTH_MICROSOFT_SECRET` environment variables
- SAML 2.0 providers (v2 feature) - Okta, Azure AD, OneLogin enterprise SSO
- OpenID Connect (v2 feature) - Custom identity providers

**Background Jobs & Async Operations:**
- Inngest - Event-driven serverless function platform for:
  - Sending meeting reminder emails (24 hours and 1 hour before meetings)
  - Computing analytics snapshots (nightly/weekly metric aggregation)
  - Auto-carrying over unfinished action items to next session
  - Sending post-session summary emails
  - Triggered by application events (e.g., `session.completed`) and cron jobs (e.g., `cron.daily`)
  - SDK/Client: `inngest` npm package
  - Auth: `INNGEST_EVENT_KEY`, `INNGEST_SIGNING_KEY` environment variables
  - Functions located in `src/lib/jobs/`

## Data Storage

**Databases:**
- PostgreSQL 16 (primary)
  - Connection: `DATABASE_URL` environment variable
  - Client: Drizzle ORM (`drizzle-orm` npm package)
  - Hosting: Neon or Supabase (managed PostgreSQL with branching)
  - Features used: JSONB columns (answer configs, tenant settings), Row-Level Security (RLS) policies, window functions (analytics aggregation)
  - Schema location: `src/lib/db/schema/`
  - Migrations: Auto-generated by Drizzle in `src/lib/db/migrations/`

**File Storage:**
- Cloudflare R2 or AWS S3 - Object storage for:
  - Profile pictures
  - PDF session exports
  - Attachments
  - Configuration: `R2_ACCOUNT_ID`, `R2_ACCESS_KEY`, `R2_SECRET_KEY`, `R2_BUCKET` environment variables
  - Client implementation expected in `src/lib/storage/` or similar

**Caching:**
- None at application level
- TanStack Query (React Query) provides client-side caching and background refetching

## Authentication & Identity

**Auth Provider:**
- Auth.js v5 (NextAuth) - Session-based authentication with support for:
  - Email + password (credential-based, bcrypt hashing with cost factor 12)
  - Magic link passwordless login
  - OAuth 2.0 (Google, Microsoft)
  - SAML 2.0 (v2 feature)
  - OpenID Connect (v2 feature)
  - Implementation: `src/lib/auth/config.ts`
  - Session configuration: HTTP-only secure cookies with SameSite=Strict
  - Session expiry: Configurable per tenant (default 24 hours, max 30 days)

**JWT Tokens (API authentication):**
- Short-lived access tokens (15 minutes) + refresh tokens for API authentication
- Implementation expected in `src/lib/auth/`

**Password Requirements:**
- Minimum 8 characters
- Must contain uppercase, lowercase, and numeric characters
- Password reset via time-limited magic link (1 hour expiry)

**Middleware & Route Protection:**
- Next.js middleware for auth redirect and session validation
- Location: `src/middleware.ts`
- Route protection implemented in `src/lib/auth/middleware.ts`

## Environment Configuration

**Required env vars:**
- `DATABASE_URL` - PostgreSQL connection string (e.g., `postgresql://user:pass@host/db`)
- `AUTH_SECRET` - Encryption key for Auth.js sessions
- `AUTH_GOOGLE_ID`, `AUTH_GOOGLE_SECRET` - Google OAuth credentials
- `AUTH_MICROSOFT_ID`, `AUTH_MICROSOFT_SECRET` - Microsoft OAuth credentials
- `RESEND_API_KEY` - Resend email service API key
- `R2_ACCOUNT_ID`, `R2_ACCESS_KEY`, `R2_SECRET_KEY`, `R2_BUCKET` - Cloudflare R2 file storage
- `INNGEST_EVENT_KEY`, `INNGEST_SIGNING_KEY` - Inngest background jobs credentials
- `NEXT_PUBLIC_APP_URL` - Public application URL (e.g., `https://app.1on1.example.com`)

**Secrets location:**
- `.env.local` for development (git-ignored)
- `.env.example` for template of required variables
- Vercel environment variables UI for production secrets management

## Integration Architecture Patterns

**Multi-tenancy with External Services:**
- Tenant isolation enforced at application level via `tenant_id` filtering
- External services (Resend, Inngest) receive tenant context via encrypted/signed payloads
- Private note encryption: AES-256-GCM with per-tenant keys derived from master key via HKDF
- File storage paths namespaced by `tenant_id` to prevent cross-tenant data leakage

**Async Job Orchestration:**
- Events published to Inngest from API routes (`src/app/api/`)
- Event types: `session.completed`, `session.scheduled`, `cron.daily`, `cron.weekly`, etc.
- Job functions handle email sends, analytics computation, and action item management
- Implemented in `src/lib/jobs/`
