**Step 6: Create organization settings API route** `src/app/api/settings/company/route.ts`:

This is the write path (mutations go through API routes per project architecture).

**GET handler:**
- Read session with `auth()` -- return 401 if not authenticated
- Check `session.user.role === "admin"` -- return 403 if not admin
- Use `withTenantContext(session.user.tenantId, session.user.id, ...)` to fetch the tenant record
- Return tenant's settings, name, slug, orgType as JSON

**PUT handler:**
- Read session -- 401 if not authenticated, 403 if not admin
- Parse request body with `orgSettingsSchema` (Zod validation)
- Use `withTenantContext()` to update the tenant's `settings` JSONB field
- Also accept optional `name` field update (org name change)
- Return updated tenant data
- **IMPORTANT**: The settings JSONB is the ONLY field being updated via this endpoint. The `orgType` is set at registration and NOT changeable via settings (it's a structural property). The `slug` is also not changeable via settings.
