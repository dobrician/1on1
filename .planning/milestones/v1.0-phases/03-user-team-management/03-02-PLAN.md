**Step 1: Create invite email template** at `src/lib/email/templates/invite.tsx`:
- Follow the pattern from existing `src/lib/email/templates/verification.tsx` for styling
- Props: inviteUrl, organizationName, inviterName, role
- Content: "You've been invited to join {organizationName} as a {role}" with Accept Invitation button
- Footer: "This invitation expires in 7 days..."
- Follow exact template from 03-RESEARCH.md "Invite Email Template"
- Reuse the same inline style objects from verification.tsx (body, container, section, heading, paragraph, button, hr, footer)

**Step 2: Create POST /api/invites/route.ts** (send invites):
- Auth check: `const session = await auth(); if (!session?.user) return 401`
- Role check: `requireRole(session.user.role, "admin")` -- only admins can invite
- Parse body with `inviteUsersSchema`
- For each email in the parsed array:
  - Check if user with this email already exists in tenant (skip with message)
  - Check if active invite already exists for this email in tenant (skip with message)
  - Generate secure token: `randomBytes(32).toString("hex")` (import from crypto)
  - Set expiresAt to 7 days from now
  - Inside `withTenantContext`: insert invite_token row AND log audit event "invite_sent"
  - Get tenant name from session or query tenants table for the email
  - Render InviteEmail and send via nodemailer (follow sendVerificationEmail pattern)
  - The invite URL: `${baseUrl}/invite/${token}` where baseUrl is derived from request headers (same pattern as existing email functions)
- Return JSON: { sent: number, skipped: [{ email, reason }] }
- Handle unique constraint violations gracefully (race condition with "already invited")

**Step 3: Create POST /api/invites/accept/route.ts** (accept invite -- PUBLIC, no auth required):
- This is a PUBLIC endpoint (no auth check -- invitees are not logged in yet)
- Parse body with `acceptInviteSchema`
- Look up invite_token by token value
- Validate: token exists, not already accepted, not expired
- If invalid/expired: return 400 with appropriate message
- Hash password with bcryptjs (import bcrypt from "bcryptjs", hash with cost 12)
- Use adminDb (not withTenantContext since user is unauthenticated) or raw db transaction:
  - Insert new user into users table: tenantId from invite, email from invite, firstName, lastName, jobTitle, passwordHash, role from invite, invitedAt (from invite.createdAt), inviteAcceptedAt (now), isActive true
  - Update invite_token: set acceptedAt to now
  - Log audit event "invite_accepted" (use the new user's ID as actorId)
- Return JSON: { success: true, email: invite.email }
- NOTE: The accept route uses the Neon adminDb directly (same as registration flow in auth/actions.ts) since there's no session context. Set tenant context via SQL `SET LOCAL` manually in the transaction.

**Step 4: Create POST /api/invites/resend/route.ts** (resend invite):
- Auth check + requireRole "admin"
- Body: { email: string } (validate with z.object({ email: z.string().email() }))
- Look up existing invite for this email in tenant
- If no invite found or invite already accepted: return 400
- Generate new token and new expiresAt (7 days)
- Inside withTenantContext: update invite_token with new token/expiresAt + log "invite_resent"
- Send new invite email
- Return JSON: { success: true }
