**Step 1: Create GET/POST /api/teams/route.ts** (list + create):

**GET (list teams):**
- Auth check required. All roles can list teams.
- Query teams within tenant, include member count (via subquery or join), include manager (team lead) info
- Return: `{ id, name, description, managerId, managerName, managerAvatarUrl, memberCount, createdAt }[]`

**POST (create team):**
- Auth check + canManageTeams check (admin or manager only)
- Parse with createTeamSchema
- Inside withTenantContext:
  - Insert team with tenantId from session
  - If managerId provided: also insert a teamMembers row with role="lead" for that user
  - Log "team_created" audit event with { teamName }
- Return created team

**Step 2: Create GET/PATCH/DELETE /api/teams/[id]/route.ts:**

**GET (team detail):**
- Auth check. All roles can view.
- Fetch team by ID within tenant
- Include full member list: join teamMembers + users to get { userId, firstName, lastName, email, avatarUrl, role (lead/member), joinedAt }
- Return 404 if not found

**PATCH (update team):**
- Auth check + canManageTeams
- Parse with updateTeamSchema
- Inside withTenantContext:
  - Update team name/description if provided
  - If managerId changed: update the old lead's teamMember role to "member", add new lead as teamMember with role="lead" (or update existing membership). Log "team_lead_changed" with { previousLeadId, newLeadId }.
  - Log "team_updated" if name/description changed
  - Set updatedAt to now()
- Return updated team

**DELETE (delete team):**
- Auth check + requireRole "admin" (only admins can delete teams)
- Inside withTenantContext:
  - Delete all teamMembers for this team
  - Delete the team
  - Log "team_deleted" with { teamName }
- Return { success: true }

**Step 3: Create POST/DELETE /api/teams/[id]/members/route.ts:**

**POST (add members):**
- Auth check + canManageTeams
- Parse with addTeamMembersSchema (array of userIds)
- Inside withTenantContext:
  - For each userId: verify user exists in tenant and is active
  - Insert teamMembers rows with role="member" (use onConflictDoNothing for already-existing members)
  - Log "member_added_to_team" for each new member with { userId, teamId }
- Return updated member list

**DELETE (remove member):**
- Auth check + canManageTeams
- Parse with removeTeamMemberSchema (single userId)
- Inside withTenantContext:
  - Delete teamMember row for userId + teamId
  - Log "member_removed_from_team" with { userId, teamId }
- Return { success: true }
