    **1. Schema migration -- add is_archived to questionnaire_template:**
    - In `src/lib/db/schema/templates.ts`, add `isArchived: boolean("is_archived").notNull().default(false)` to the `questionnaireTemplates` table definition (after `version`).
    - Run `bunx drizzle-kit generate` to create the migration SQL.
    - Apply the migration locally via `psql` (same pattern as Phase 1 -- Neon driver can't connect locally): `psql -h localhost -U postgres -d oneononestaging -f src/lib/db/migrations/{generated_file}.sql` and also for `oneononeprod` if blue-green is active. Check the generated SQL and apply appropriately.

    **2. Zod validation schemas -- `src/lib/validations/template.ts`:**
    Follow the pattern from `src/lib/validations/team.ts`. Create these schemas:
    - `createTemplateSchema`: name (string 1-255, required), description (string max 2000, optional), category (enum: check_in, career, performance, onboarding, custom)
    - `updateTemplateSchema`: same fields but all optional
    - `questionSchema`: questionText (string 1-1000), helpText (string max 500, nullable optional), category (enum: check_in, wellbeing, engagement, performance, career, feedback, recognition, goals, custom), answerType (enum: text, rating_1_5, rating_1_10, yes_no, multiple_choice, mood), answerConfig (record, default {}), isRequired (boolean, default false), sortOrder (int min 0), conditionalOnQuestionId (uuid nullable optional), conditionalOperator (enum: eq, neq, lt, gt, lte, gte, nullable optional), conditionalValue (string max 255, nullable optional)
    - `saveTemplateSchema`: name + description + category + questions array (min 1 question)
    - `reorderQuestionsSchema`: questionIds array of uuid (min 1)
    - Export the template category and question category const arrays for reuse in UI dropdowns.

    **3. RBAC helper -- add `canManageTemplates` to `src/lib/auth/rbac.ts`:**
    Add `export function canManageTemplates(role: string): boolean { return role === "admin" || role === "manager"; }` -- same pattern as canManageTeams.

    **4. API routes -- follow exact pattern from `src/app/api/teams/route.ts`:**

    **GET /api/templates** (`src/app/api/templates/route.ts`):
    - Auth check, withTenantContext
    - Select templates with LEFT JOIN to template_questions (where is_archived=false) for question count
    - Filter out archived templates (is_archived=false) by default, accept `?include_archived=true` query param
    - Return: id, name, description, category, isDefault, isPublished, isArchived, version, createdAt, questionCount
    - Order by name

    **POST /api/templates** (`src/app/api/templates/route.ts`):
    - Auth + canManageTemplates RBAC check
    - Validate with createTemplateSchema
    - Insert into questionnaireTemplates with tenantId, createdBy from session
    - logAuditEvent: action "template_created", resourceType "template"
    - Return 201 with template

    **GET /api/templates/[id]** (`src/app/api/templates/[id]/route.ts`):
    - Auth check, withTenantContext
    - Fetch template by id
    - Fetch all non-archived questions for this template ordered by sortOrder
    - Return template with questions array

    **PATCH /api/templates/[id]** (`src/app/api/templates/[id]/route.ts`):
    - Auth + canManageTemplates check
    - Validate with updateTemplateSchema
    - Update template metadata (name, description, category)
    - Set updatedAt to new Date()
    - logAuditEvent: action "template_updated"

    **DELETE /api/templates/[id]** (`src/app/api/templates/[id]/route.ts`):
    - This is a soft-delete (archive). Set is_archived=true, NOT actual deletion
    - Auth + canManageTemplates check
    - If template is_default, unset it first
    - logAuditEvent: action "template_archived"
    - Return 200

    **POST /api/templates/[id]/questions** (`src/app/api/templates/[id]/questions/route.ts`):
    - Auth + canManageTemplates check
    - Validate with questionSchema
    - Validate answer_config matches answerType (multiple_choice must have options array with min 2 items)
    - Insert question with templateId
    - Return 201 with question

    **PATCH /api/templates/[id]/questions/[questionId]** (`src/app/api/templates/[id]/questions/[questionId]/route.ts`):
    - Auth + canManageTemplates check
    - Validate with partial questionSchema
    - Update question fields
    - Return updated question

    **DELETE /api/templates/[id]/questions/[questionId]** (`src/app/api/templates/[id]/questions/[questionId]/route.ts`):
    - Soft delete: set is_archived=true on the question (never actually delete)
    - Auth + canManageTemplates check
