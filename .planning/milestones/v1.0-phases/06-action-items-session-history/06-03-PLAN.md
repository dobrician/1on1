1. Create a new SQL migration for GIN expression indexes. Use `npx drizzle-kit generate` to create a blank migration file, then add the following SQL (or create a custom migration file in the migrations directory):

   ```sql
   -- GIN indexes for full-text search
   CREATE INDEX IF NOT EXISTS action_item_search_idx ON action_item
     USING gin(to_tsvector('english', coalesce(title, '') || ' ' || coalesce(description, '')));

   CREATE INDEX IF NOT EXISTS talking_point_search_idx ON talking_point
     USING gin(to_tsvector('english', content));

   CREATE INDEX IF NOT EXISTS session_answer_text_search_idx ON session_answer
     USING gin(to_tsvector('english', coalesce(answer_text, '')));
   ```

   Note: shared_notes is JSONB so GIN indexing on extracted text is complex. Use on-the-fly extraction for shared_notes search queries -- at v1 data volumes this is fine without an index.

   Apply the migration locally via Docker exec psql (same pattern as Phase 1):
   ```bash
   docker exec -i 1on1-db-1 psql -U postgres -d oneononev2 < src/lib/db/migrations/NNNN_full_text_search_indexes.sql
   ```

2. Create `src/app/api/search/route.ts` with GET handler:
   - Auth check (401)
   - Accept query params: q (search query string, required), limit (default 5 per type, max 10)
   - Validate: if q is empty or whitespace-only, return `{ results: { sessions: [], actionItems: [], templates: [], people: [] } }`
   - Sanitize input: trim whitespace. The `websearch_to_tsquery` function handles natural language safely.
   - Uses withTenantContext to run parallel search queries:

   a. **Session search** (searches talking points, free-text answers, and shared notes):
      - Talking points: `to_tsvector('english', content) @@ websearch_to_tsquery('english', $q)`
        JOIN sessions + meetingSeries, filter by participant visibility
        Use `ts_headline('english', content, websearch_to_tsquery('english', $q), 'MaxWords=20,MinWords=10,MaxFragments=1')` for snippet
        Return: sessionId, sessionNumber, snippet, seriesId, reportName, scheduledAt
      - Free-text answers: same pattern on session_answer.answer_text WHERE answer_text IS NOT NULL
      - Shared notes: extract JSONB text values with `(SELECT string_agg(value::text, ' ') FROM jsonb_each_text(session.shared_notes))`, then to_tsvector on the extracted text. This is on-the-fly (no index), acceptable for v1.
      - Deduplicate by sessionId -- if multiple matches in same session, keep highest rank, merge snippets
      - Limit to `limit` results

   b. **Action items search**:
      - `to_tsvector('english', coalesce(title, '') || ' ' || coalesce(description, '')) @@ websearch_to_tsquery('english', $q)`
      - JOIN sessions + meetingSeries for visibility
      - Use ts_headline on title + description for snippet
      - Return: id, title, snippet, status, sessionId, seriesId
      - Limit to `limit` results

   c. **Template search** (simple ILIKE or tsvector on name + description):
      - Filter by tenantId (user's templates + system templates where tenant_id IS NULL)
      - Return: id, name, snippet (from description)
      - Limit to `limit` results

   d. **People search** (ILIKE on first_name, last_name, email, job_title):
      - Filter by tenantId
      - Return: id, firstName, lastName, email, jobTitle
      - Limit to `limit` results

   - Combine all results into grouped response:
     ```json
     {
       "results": {
         "sessions": [...],
         "actionItems": [...],
         "templates": [...],
         "people": [...]
       }
     }
     ```
   - Role-based visibility enforced via the JOIN to meetingSeries and participant check (managerId/reportId or admin)
