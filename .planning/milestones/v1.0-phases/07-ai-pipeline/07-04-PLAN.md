     - Update the `upcoming=true` handler to INCLUDE nudges with NULL targetSessionAt. Currently, `gte(aiNudges.targetSessionAt, new Date())` excludes NULLs (PostgreSQL NULL comparisons are always false).
     - Replace the upcoming filter logic with:
       ```typescript
       if (upcoming) {
         const in7Days = new Date(Date.now() + 7 * 24 * 60 * 60 * 1000);
         // Include nudges with NULL targetSessionAt OR targetSessionAt within next 7 days
         conditions.push(
           sql`(${aiNudges.targetSessionAt} IS NULL OR (${aiNudges.targetSessionAt} >= now() AND ${aiNudges.targetSessionAt} <= ${in7Days}))`
         );
       }
       ```
     - This ensures that when `upcoming=true` IS used (e.g., from other consumers), it still includes NULL-dated nudges. Remove the separate `gte` and `lte` condition pushes and replace with the single SQL expression above.

  Both changes work together: the wizard no longer requests `upcoming=true`, so it gets ALL non-dismissed nudges for the series. The API fix is defensive -- if any future consumer uses `upcoming=true`, NULLs won't be silently dropped.
