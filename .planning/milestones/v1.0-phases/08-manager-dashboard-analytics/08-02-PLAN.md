<task type="auto">
  <name>Task 1: Dashboard query layer</name>
  <files>src/lib/queries/dashboard.ts</files>
  <action>
Create `src/lib/queries/dashboard.ts` with all dashboard data queries. All queries accept TransactionClient from withTenantContext.

1. `getUpcomingSessions(tx, userId, role, tenantId)`:
   - Query sessions WHERE scheduledAt is within next 7 days AND status IN ('scheduled', 'in_progress')
   - JOIN meetingSeries for manager/report info, JOIN users for names (both manager and report)
   - For managers: filter by managerId = userId. For admins: all sessions. For members: filter by reportId = userId
   - Also fetch nudges for each session's series (batch: single query for all relevant series IDs, not N+1)
   - Return array with: sessionId, seriesId, reportName (or managerName for members), scheduledAt, status, templateName, nudges array, isToday flag
   - Sort by scheduledAt ASC

2. `getOverdueActionItems(tx, userId, role)`:
   - Query action_items WHERE status IN ('open', 'in_progress') AND dueDate < now()
   - JOIN sessions for series context, JOIN users for assignee info, JOIN meetingSeries + users for report name
   - For managers: items where series.managerId = userId. For admins: all. For members: assigneeId = userId
   - Group results by report name in the return type
   - Return: `Array<{ reportName: string; reportId: string; items: Array<{ id, title, dueDate, daysOverdue: number }> }>`

3. `getQuickStats(tx, userId, role)`:
   - Total reports: COUNT DISTINCT reportId from meetingSeries WHERE managerId = userId AND status = 'active'
   - Sessions this month: COUNT sessions WHERE completedAt in current month
   - Average score: AVG sessionScore for completed sessions this month
   - For members: sessions where they are the report. For admins: org-wide
   - Return: `{ totalReports: number; sessionsThisMonth: number; avgScore: number | null }`

4. `getRecentSessions(tx, userId, role, limit = 5)`:
   - Query sessions WHERE status = 'completed', ordered by completedAt DESC, limit
   - JOIN meetingSeries + users for report/manager names
   - Include sessionScore and aiSummary snippet (first sentence of narrative if exists)
   - Return: `Array<{ id, seriesId, reportName, completedAt, sessionScore, aiSummarySnippet: string | null }>`
  </action>
