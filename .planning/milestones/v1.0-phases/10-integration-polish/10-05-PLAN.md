    **1. Create critical path E2E test (`e2e/critical-path.spec.ts`):**

    Write a Playwright test that exercises the complete user journey. Use the existing auth setup (admin user from `auth.setup.ts`). The test should be a single describe block with sequential test steps:

    a. **Navigate to dashboard** -- verify overview page loads, shows expected elements (upcoming sessions section, quick stats).
    b. **Create a questionnaire template** -- go to Templates page, click "Create", fill in name/description, add at least 2 questions (one rating, one free text), save. Verify template appears in list.
    c. **Create a meeting series** -- go to Sessions page, click "Create Series", select a report user (from seed data), set cadence to weekly, select the template just created, save. Verify series card appears.
    d. **Start a session** -- click "Start Session" on the series card. Verify wizard loads with the correct questions.
    e. **Answer questions** -- fill in answers for each question (type in text, select rating). Verify auto-save (wait for save indicator).
    f. **Complete session** -- navigate to summary step, verify all answers displayed. Click "Complete Session". Verify redirect to session summary page.
    g. **Verify AI summary** -- on the session summary page, wait for AI summary to appear (poll with retries, up to 30 seconds). If AI is not configured in test env, check that the AI status field exists (even if "pending" or "failed").
    h. **Check dashboard** -- navigate back to overview. Verify the completed session appears in "Recent Sessions". Verify quick stats updated.
    i. **Check notifications table** -- verify a notification record was created for the session (query-based check via API or just verify the notification UI element).

    Use generous timeouts for AI-dependent steps (30s). Use `test.setTimeout(120000)` for the full critical path test.

    **4. Run tests and fix any failures:**
    - Run `bunx playwright test e2e/critical-path.spec.ts e2e/dark-mode.spec.ts`.
    - If tests fail due to application bugs (not test issues), fix the application code. This is the verification phase -- issues found here MUST be resolved.
    - Common issues to watch for: missing loading states causing flaky selectors, hardcoded URLs, race conditions in auto-save, missing ARIA labels for test selectors.
