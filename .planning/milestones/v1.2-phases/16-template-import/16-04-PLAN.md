    **handleClose:** resets all state → step "select", clears payload, errors, language mismatch, conflict, created IDs.

    **File reading (step "select"):**
    - Hidden `<input type="file" accept=".json" ref={fileInputRef} onChange={handleFileChange} className="hidden" />`
    - "Browse" button: `onClick={() => fileInputRef.current?.click()}` — do NOT wrap Button inside a label (nested interactive element pitfall)
    - handleFileChange reads file with FileReader:
      ```typescript
      const file = e.target.files?.[0];
      if (!file) return;
      setFileName(file.name);
      const reader = new FileReader();
      reader.onload = (ev) => {
        const text = ev.target?.result as string;
        let raw: unknown;
        try { raw = JSON.parse(text); }
        catch {
          setImportErrors([{ path: "root", message: t("import.error.notJson") }]);
          setStep("error");
          return;
        }
        // schemaVersion early check
        if (typeof raw === "object" && raw !== null && "schemaVersion" in raw) {
          const v = (raw as Record<string, unknown>).schemaVersion;
          if (v !== 1) {
            setImportErrors([{ path: "schemaVersion", message: t("import.schemaVersionUnsupported", { version: String(v) }) }]);
            setStep("error");
            return;
          }
        }
        const result = templateImportSchema.safeParse(raw);
        if (!result.success) {
          setImportErrors(formatImportErrors(result.error));
          setStep("error");
          return;
        }
        const payload = result.data;
        setParsedPayload(payload);
        setImportName(payload.name);
        // Language mismatch check
        const mismatch = payload.language !== contentLanguage;
        setLanguageMismatch(mismatch);
        setMismatchConfirmed(!mismatch); // if no mismatch, already "confirmed"
        setStep("preview");
      };
      reader.readAsText(file);
      ```

    **Step "preview":**
    - Show `derivePreviewStats(parsedPayload)` results: name, description, sectionCount, questionCount, typeCounts as `<Badge>` items
    - If `languageMismatch && !mismatchConfirmed`: show yellow Alert (use `border-yellow-500 bg-yellow-50 text-yellow-800` if `variant="warning"` not available in shadcn Alert). Alert body: `t("import.languageMismatch.message", { fileLanguage: parsedPayload.language, companyLanguage: contentLanguage })`. Two buttons: "Proceed anyway" (sets mismatchConfirmed: true) and "Cancel" (handleClose).
    - When mismatch present: disable "Import" button until mismatch confirmed. Show the Alert and Import/Cancel buttons.
    - When no mismatch or mismatch confirmed: show Import and Cancel buttons.
    - "Import" onClick triggers `handlePreviewConfirm()`.

    **handlePreviewConfirm:**
    - Check TanStack Query cache for name conflict: `templates?.some(t => t.name.toLowerCase() === importName.toLowerCase())`
    - If conflict: set renameValue to `importName + " (imported)"`, setStep("conflict")
    - If no conflict: trigger the import mutation

    **Step "conflict":**
    - Show: `t("import.step3.message", { name: importName })`
    - "Rename" option: inline `<Input value={renameValue} onChange={...} />` + "Import as '{renameValue}'" button → sets importName to renameValue, triggers mutation
    - "Create as copy" button → sets importName to `importName + " (copy)"`, triggers mutation immediately
    - "Cancel" button → handleClose

    **Import mutation (useMutation):**
    ```typescript
    const importMutation = useMutation({
      mutationFn: async ({ payload, name }: { payload: TemplateImportPayload; name: string }) => {
        const res = await fetch("/api/templates/import", {
          method: "POST",
          headers: { "Content-Type": "application/json" }, // REQUIRED — pitfall
          body: JSON.stringify({ payload, importName: name }),
        });
        const data = await res.json();
        if (res.status === 422) {
          throw { type: "validation", errors: data.errors as ImportError[] };
        }
        if (res.status === 409) {
          throw { type: "conflict", name: data.name as string };
        }
        if (!res.ok) {
          throw { type: "generic", message: data.error ?? t("import.failed") };
        }
        return data as { templateId: string; name: string };
      },
      onSuccess: (data) => {
        setCreatedId(data.templateId);
        setCreatedName(data.name);
        setStep("success");
        queryClient.invalidateQueries({ queryKey: ["templates"] });
        onImportSuccess?.();
      },
      onError: (error: unknown) => {
        const err = error as { type: string; errors?: ImportError[]; name?: string; message?: string };
        if (err.type === "validation") {
          setImportErrors(err.errors ?? []);
          setStep("error");
        } else if (err.type === "conflict") {
          // Server-side conflict (race condition) — show conflict step with server-returned name
          setRenameValue((err.name ?? importName) + " (imported)");
          setStep("conflict");
        } else {
          toast.error(err.message ?? t("import.failed"));
        }
      },
    });
    ```

    **Step "error":**
    - Title: `t("import.error.title")`
    - Show up to 10 errors from `importErrors`:
      ```tsx
      <ul className="max-h-48 overflow-y-auto space-y-1 text-sm">
        {importErrors.slice(0, 10).map((e, i) => (
          <li key={i}><span className="font-mono text-xs text-muted-foreground">{e.path}:</span> {e.message}</li>
        ))}
        {importErrors.length > 10 && (
          <li className="text-muted-foreground">{t("import.error.tooManyErrors", { count: importErrors.length - 10 })}</li>
        )}
      </ul>
      ```
    - "Close" button only — no proceed option for invalid files

    **Step "success":**

    - Message: `t("import.step4.message", { name: createdName })`
