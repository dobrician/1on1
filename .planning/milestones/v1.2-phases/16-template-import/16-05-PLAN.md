  <how-to-verify>
    Start dev server if not running: `bun run dev --port 4300`
    Open: https://1on1.surmont.co/templates

    **Test 1 — RBAC (IMP-01):**
    - Log in as admin → confirm "Import" button visible in header next to "New Template"
    - Log in as manager → confirm "Import" button visible
    - Log in as member → confirm NO "Import" button visible

    **Test 2 — Happy path round-trip (IMP-01, IMP-02):**
    - As admin, click "Export" on any existing template to download a .json file
    - Click "Import" → dialog opens on step 1 (file select)
    - Click "Browse" → select the downloaded .json file
    - Click "Parse & Preview" → step 2 shows: template name, section count, question count, type breakdown badges
    - Confirm counts match the exported template's actual structure
    - Click "Import" → if name conflict appears (step 3): click "Create as copy" → proceeds
    - Success step shows template name + "View Template" link
    - Click "View Template" → navigates to the newly imported template
    - Template appears in the template list

    **Test 3 — Language mismatch warning (IMP-03):**
    - Export a template from a company account where content language = "en"
    - Switch to a company account where content language = "ro" (or change company language in settings)
    - Import the "en" template → Preview step shows a yellow warning banner:
      "This template is in en but your company uses ro" (or localized version)
    - "Import" button is disabled until "Proceed anyway" is clicked
    - Click "Proceed anyway" → Import button becomes active → proceed with import

    **Test 4 — Name conflict resolution (IMP-04):**
    - Import a template that already has a template with the same name
    - Confirm the conflict step (step 3) appears with: existing name, Rename input, "Create as copy" button, Cancel
    - Test Rename: change the name in the input → click "Import as [name]" → success step
    - Test Create as copy: trigger conflict again with same name → click "Create as copy" → success step with "(copy)" suffix

    **Test 5 — Invalid file, field-specific errors (IMP-05):**
    - Create a test file `invalid.json` with content: `{"schemaVersion":1,"language":"en","name":"","sections":[{"name":"S","sortOrder":0,"description":null,"questions":[{"questionText":"Q","helpText":null,"answerType":"checkbox","isRequired":true,"sortOrder":0,"scoreWeight":1,"conditionalOnQuestionSortOrder":null,"conditionalOperator":null,"conditionalValue":null,"answerConfig":{}}]}]}`
    - Import this file → error step shows:
      - Error for empty name: `field \`name\`:`
      - Error for invalid answerType: `Section 1, Question 1, field \`answerType\`:`
    - "Close" button only — no proceed option

    **Test 6 — Reset on close:**
    - Open import dialog, upload a file to reach preview step
    - Click X to close dialog
    - Reopen dialog → step 1 (file select) is shown, no previous state visible
