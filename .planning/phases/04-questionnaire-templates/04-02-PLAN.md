  <action>
    **1. Versioning-aware PATCH -- update `src/app/api/templates/[id]/route.ts`:**
    Enhance the existing PATCH handler to support batch saves with versioning:
    - Accept the `saveTemplateSchema` body (template metadata + full questions array)
    - Check if the template has been used in any session: `SELECT count(*) FROM session WHERE template_id = ?`
      - Note: the `sessions` table exists in schema (from Phase 1). If no sessions reference this template, update in place.
      - If sessions reference this template AND questions have changed: increment `version`, archive all current non-archived questions (set is_archived=true), insert new question rows with new UUIDs
    - For unused templates: update metadata and questions directly (upsert pattern -- update existing, insert new, archive removed)
    - Always update `updatedAt` to now
    - logAuditEvent with version info in metadata

    **2. POST /api/templates/[id]/duplicate -- `src/app/api/templates/[id]/duplicate/route.ts`:**
    - Auth + canManageTemplates check
    - Fetch the source template + non-archived questions
    - Create new template: name = "{original name} (Copy)", same description/category, version=1, isDefault=false, isPublished=false, isArchived=false
    - Copy all non-archived questions with NEW UUIDs and the new templateId
    - CRITICAL: Build an oldId->newId map. For any question with conditionalOnQuestionId, remap it to the new UUID using the map. This prevents the duplicate from referencing the original's question IDs.
    - logAuditEvent: action "template_duplicated", metadata includes sourceTemplateId
    - Return 201 with the new template (including questions)

    **3. PUT /api/templates/[id]/default -- `src/app/api/templates/[id]/default/route.ts`:**
    - Auth + isAdmin check (only admins can set org default, not managers)
    - In a SINGLE transaction (withTenantContext already provides this):
      1. `UPDATE questionnaire_template SET is_default=false WHERE tenant_id=? AND is_default=true` (unset all current defaults)
      2. `UPDATE questionnaire_template SET is_default=true WHERE id=?` (set new default)
    - This prevents race conditions with two admins setting defaults simultaneously
    - logAuditEvent: action "template_set_default"
    - Return 200

    **4. PUT /api/templates/[id]/publish -- `src/app/api/templates/[id]/publish/route.ts`:**
    - Auth + canManageTemplates check
    - Toggle isPublished: read current value, set opposite
    - Only allow publishing if template has at least 1 non-archived question
    - logAuditEvent: action "template_published" or "template_unpublished"
    - Return 200 with { isPublished: boolean }
  </action>
