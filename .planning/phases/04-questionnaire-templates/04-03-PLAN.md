    **1. ConditionalLogicForm component -- `src/components/templates/conditional-logic-form.tsx`:**
    "use client" component that configures show/hide conditions on a question:
    - **Props:** currentQuestionIndex (number), allQuestions (array), onConditionChange (callback with {conditionalOnQuestionId, conditionalOperator, conditionalValue} | null)
    - **Enable toggle:** Checkbox or Switch "Show this question conditionally". When unchecked, all conditional fields are null.
    - **When enabled, show 3 fields:**
      1. **"When question..."** -- Select dropdown listing only questions with sortOrder LOWER than the current question. Display format: "Q{n}: {truncated question text}". This prevents circular references by construction.
      2. **"...is"** -- Select dropdown with operators. Filter operators by the selected question's answer type:
         - text: only eq, neq (no numeric comparisons on text)
         - rating_1_5, rating_1_10: all 6 operators (eq, neq, lt, gt, lte, gte)
         - yes_no: only eq, neq (values: "true", "false")
         - multiple_choice: only eq, neq (value = one of the option strings)
         - mood: all 6 operators (mood maps to numeric 1-5)
      3. **"...value"** -- Input that adapts to the selected question's answer type:
         - text: text input
         - rating_1_5: select with options 1-5
         - rating_1_10: select with options 1-10
         - yes_no: select with Yes/No
         - multiple_choice: select with the target question's options from answer_config
         - mood: select with 1-5 (or emoji labels if configured)
    - **Validation:** All three fields must be filled when conditional is enabled. Show inline error if incomplete.

    **2. Integrate into QuestionForm (`src/components/templates/question-form.tsx`):**
    - Add ConditionalLogicForm at the bottom of the question form dialog
    - Pass the current question's index in the questions array and the full questions list
    - When condition changes, update the form's conditionalOnQuestionId, conditionalOperator, conditionalValue fields
    - For new questions (index = last), they can reference any existing question

    **3. Update QuestionCard (`src/components/templates/question-card.tsx`):**
    - If question has conditionalOnQuestionId set, show a small indicator below the question:
      - "Shows when Q{n} {operator} {value}" in muted text with a conditional icon (e.g., GitBranch from lucide-react)
    - This gives at-a-glance visibility into which questions are conditional without opening the edit form

    **4. Server-side validation -- ensure template save validates conditional logic:**
    - The PATCH handler in /api/templates/[id]/route.ts should validate:
      - conditionalOnQuestionId references a question in the same template
      - The referenced question has a lower sortOrder (earlier in the list)
      - The operator is valid for the referenced question's answer type
    - Add a `validateConditionalLogic(questions)` utility function in `src/lib/validations/template.ts`:
      - Input: array of questions with their conditional fields
      - Returns: null if valid, or error string describing the problem
      - Checks: target exists, target is earlier, operator compatible with answer type, both operator and value present when conditionalOnQuestionId is set
    - Call this function in the PATCH handler before saving. Return 400 with the error message if invalid.
