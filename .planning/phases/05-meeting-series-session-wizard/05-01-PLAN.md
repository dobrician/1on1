    **API Routes (follow exact pattern from templates API: auth -> RBAC -> Zod -> withTenantContext -> audit):**

    1. `GET /api/series` -- List all series for the tenant. JOIN with users table to get report's firstName, lastName, avatarUrl. JOIN with sessions to get the latest session status/number. Filter by tenant_id. Allow `?status=active` query param. Order by nextSessionAt ASC (soonest first, nulls last).
    2. `POST /api/series` -- Create series. Validate with createSeriesSchema. Verify the reportId is a user in the same tenant. Verify no existing active series for the same manager+report pair (unique constraint will catch this too). Compute nextSessionAt using computeNextSessionDate(new Date(), cadence, cadenceCustomDays, preferredDay). Audit log: "series_created". The managerId comes from session.user.id (the creator is the manager).
    3. `GET /api/series/[id]` -- Get series detail with settings + sessions history. JOIN with users for manager/report info. Include sessions ordered by sessionNumber DESC with their scores and statuses.
    4. `PATCH /api/series/[id]` -- Update series settings (cadence, template, preferred day/time, status). Validate with updateSeriesSchema. Recompute nextSessionAt if cadence changes. Audit log: "series_updated".
    5. `DELETE /api/series/[id]` -- Archive series (soft delete: set status to "archived"). Audit log: "series_archived".
    6. `POST /api/series/[id]/start` -- Start a new session. Creates session record with status "in_progress", sessionNumber (count of existing sessions + 1), templateId from series.defaultTemplateId, scheduledAt = now(), startedAt = now(). Returns the session ID. Only the manager can start a session. Audit log: "session_started".

    **UI Components:**

Card with report avatar (Avatar/AvatarFallback), report name as title, cadence as subtitle (capitalize), nextSessionAt formatted as relative date, status Badge (active=default, paused=secondary, archived=outline), Start/Resume button (variant depends on whether there's an in_progress session for this series). Link the card itself to /sessions/[id].
    8. `series-list.tsx` -- Client component. Receives series as props from Server Component, uses useQuery for refresh. Renders a responsive grid: `grid grid-cols-1 md:grid-cols-2 lg:grid-cols-3 gap-4`. Empty state: "No meeting series yet" with a "Create Series" button linking to /sessions/new.
    9. `series-form.tsx` -- Client component (used on /sessions/new page). useForm with zodResolver(createSeriesSchema). Fields: Report (select from users API -- use a combobox/select), Cadence (radio group: weekly/biweekly/monthly/custom + custom days input), Preferred Day (select), Preferred Time (time input), Default Template (select from templates API), Duration (number input with 30min default). On submit, POST /api/series, redirect to /sessions on success. Toast on error.
    10. `series-detail.tsx` -- Client component on the [id] page. Shows series settings at top (report name, cadence, next session, status with Edit/Pause/Archive buttons), session history timeline below. "Start Session" button at top.
