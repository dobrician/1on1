    **Wizard layout (`src/app/(session-wizard)/layout.tsx`):**
    Create a separate route group layout that does NOT include the Sidebar. This provides the full-page immersive experience per CONTEXT.md.
    ```
    - Import auth, SessionProvider, QueryProvider, Toaster (same providers as dashboard layout)
    - Server-side auth check: if !session redirect to /login
    - Render: SessionProvider > QueryProvider > div.flex.min-h-screen.flex-col > {children} + Toaster
    - NO Sidebar component, NO header with ThemeToggle
    ```

    **Session data API (`GET /api/sessions/[id]`):**
    Returns the full session data needed by the wizard in a single request:
    - Session record with status, sessionNumber, startedAt
    - Series info: managerId, reportId, report firstName/lastName/avatarUrl, cadence
    - Template questions: all non-archived questions for the session's templateId, ordered by sortOrder, including answerType, answerConfig, category, helpText, isRequired, conditionalOnQuestionId, conditionalOperator, conditionalValue
    - Existing answers: all session_answers for this session (to restore state on Resume)
    - Previous sessions: last 3 completed sessions for the same series, with their answers, shared_notes, and action items (for the recap and context panel)
    - Open action items: from previous sessions in the same series where status is "open" or "in_progress"
    - Authorization: user must be the manager or report on the series, or an admin. Use isSeriesParticipant from rbac.ts.

    **Answer upsert API (`PUT /api/sessions/[id]/answers`):**
    Upserts a single answer for the auto-save pattern:
    - Input: `{ questionId: string, answerText?: string, answerNumeric?: number, answerJson?: any, skipped?: boolean }`
    - Uses onConflictDoUpdate on the new UNIQUE(session_id, question_id) index
    - Sets respondentId from session.user.id
    - Returns the upserted answer
    - Authorization: user must be participant on the session's series

    **Zod validations (`src/lib/validations/session.ts`):**
    - `answerUpsertSchema`: { questionId: z.string().uuid(), answerText: z.string().optional(), answerNumeric: z.number().optional(), answerJson: z.any().optional(), skipped: z.boolean().optional() }
    - Export `AnswerUpsertInput`

    **Debounce hook (`src/lib/hooks/use-debounce.ts`):**
    Standard useDebounce<T>(value: T, delay = 500): T pattern. Uses useState + useEffect with cleanup.

    **Proxy update (`proxy.ts`):**
    Add the `/wizard` path to the authenticated routes (the wizard route group is at `(session-wizard)/wizard/[sessionId]` which resolves to `/wizard/[sessionId]` in the URL). Ensure unauthenticated users are redirected to /login when accessing /wizard/*.

    **Wizard page (`src/app/(session-wizard)/wizard/[sessionId]/page.tsx`):**
    Server Component that reads sessionId from params, renders WizardShell client component. Minimal -- delegates everything to WizardShell.

    **Wizard shell (`wizard-shell.tsx`) -- the core orchestrator:**
    "use client" component that manages all wizard state.
    - Fetches session data via useQuery from GET /api/sessions/[sessionId]
    - State management via useReducer with this shape:
      ```
      {
        currentStep: number, // 0 = recap, 1..N = categories, N+1 = summary (summary in Plan 05)
        answers: Map<questionId, { answerText, answerNumeric, answerJson, skipped }>,
        categories: Array<{ name: string, questions: TemplateQuestion[] }>,
        saveStatus: 'saved' | 'saving' | 'error',
        previousSessions: PreviousSession[],
        openActionItems: ActionItem[],
      }
      ```
    - Groups template questions by category using the pattern from RESEARCH.md: `groupQuestionsByCategory(questions)` creates one step per unique category, ordered by first question's sortOrder in that category
    - Category order is canonical: defined by the order categories appear when sorting all questions by sortOrder. This prevents ordering inconsistencies (Pitfall 5 from RESEARCH.md)
    - Steps: [recap, ...categories, summary(later)]. currentStep=0 renders RecapScreen, 1..N renders CategoryStep, N+1 renders SummaryScreen (added in Plan 05)
    - Conditional logic: evaluate ALL conditions based on current answers across ALL categories (Pitfall 6 from RESEARCH.md). Track all answers at wizard level. A question is visible if: (a) no conditional, or (b) condition evaluates to true based on the answer to conditionalOnQuestionId
    - Passes answer change handler down to widgets. On answer change: update local state immediately, trigger debounced save via useMutation to PUT /api/sessions/[id]/answers
    - Track saveStatus: set 'saving' when mutation fires, 'saved' on success, 'error' on failure
    - Add beforeunload event listener that triggers immediate save of any pending debounced values (Pitfall 3 from RESEARCH.md)
    - Layout: flex-col full height. TopBar at top, main content area in middle (scrollable), Navigation at bottom

    **Wizard top bar (`wizard-top-bar.tsx`):**

    Use absolute minimal styling. No navigation chrome. Per CONTEXT.md: "minimal top bar".
