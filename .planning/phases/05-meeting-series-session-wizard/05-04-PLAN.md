    **Notes editor (`notes-editor.tsx`):**
    "use client" component using tiptap for rich text editing.

    Props:
    ```typescript
    interface NotesEditorProps {
      sessionId: string;
      category: string;
      initialSharedContent: string; // HTML from sharedNotes JSONB[category]
      initialPrivateContent: string; // decrypted HTML from private_note
      readOnly?: boolean;
    }
    ```

    Implementation:
    - Two tabs (use shadcn Tabs): "Shared Notes" (default) and "Private Notes"
    - Each tab has its own tiptap editor instance
    - CRITICAL: Set `immediatelyRender: false` on both editors to prevent SSR hydration mismatch (Pitfall 1 from RESEARCH.md)
    - Extensions: StarterKit (bold, italic, lists, headings, blockquote) + Link.configure({ openOnClick: false })
    - Simple toolbar above editor: Bold (B), Italic (I), Bullet List, Ordered List, Link buttons. Use editor.chain().focus().toggleBold().run() pattern
    - Toolbar buttons use small icon buttons (lucide icons: Bold, Italic, List, ListOrdered, LinkIcon)
    - Private notes tab shows a small lock icon and "Only you can see this" label
    - Content auto-saves with debounce: on editor `onUpdate`, update local state. Debounced value triggers PUT to the appropriate API endpoint
    - Shared notes: PUT /api/sessions/[id]/notes with { category, content: html }
    - Private notes: PUT /api/sessions/[id]/notes/private with { category, content: html }
    - Store content as HTML string (getHTML()). Per RESEARCH.md recommendation: simpler for display, Phase 7 AI can parse HTML easily

    **Shared notes API (`PUT /api/sessions/[id]/notes`):**
    - Auth check + isSeriesParticipant authorization
    - Input: `{ category: string, content: string }`
    - Read current sharedNotes JSONB from session, update the [category] key, write back
    - Uses withTenantContext

    **Private notes API (`PUT /api/sessions/[id]/notes/private` and `GET`):**
    - Auth check + verify author is the requesting user (private notes are author-only)
    - PUT input: `{ category: string, content: string }`
    - Server-side encryption: `encryptNote(content, tenantId)` before storing. Store the EncryptedPayload JSON string in the `content` column
    - Upsert using UNIQUE(session_id, author_id, category) constraint
    - Cache the derived tenant key for the duration of the request (Pitfall 4 from RESEARCH.md)
    - GET: Read private notes for this session + author, decrypt each with `decryptNote(JSON.parse(content), tenantId)`, return decrypted content keyed by category

    **Talking points list (`talking-point-list.tsx`):**
    "use client" component.

    Props:
    ```typescript
    interface TalkingPointListProps {
      sessionId: string;
      category: string;
      initialPoints: TalkingPoint[];
    }
    ```

    Implementation:
    - List of talking points with checkbox toggle (isDiscussed)
    - "Add talking point" input at bottom -- text input with Enter to add
    - Each point: checkbox + text + delete button (X icon)
    - Checking a point sends PATCH to toggle isDiscussed (sets discussedAt)
    - Adding sends POST to /api/sessions/[id]/talking-points with { content, category, sortOrder }
    - Use useMutation for create and update, with optimistic updates
    - Carried-over points from previous sessions show a small "Carried from Session #N" badge

    **Talking points API (`/api/sessions/[id]/talking-points`):**
    - POST: Create talking point. Input: `{ content: string, category: string, sortOrder: number }`
    - PATCH: Toggle discussed. Input: `{ id: string, isDiscussed: boolean }`
    - GET: List talking points for session. Filter: optional `?category=xxx`
    - Auth: isSeriesParticipant

    **Action item inline (`action-item-inline.tsx`):**
    "use client" component.

    Props:
    ```typescript
    interface ActionItemInlineProps {
      sessionId: string;
      category: string;
    ```

    Implementation:

    - On submit: POST /api/sessions/[id]/action-items with { title, assigneeId, dueDate, category }
    - Use useMutation with optimistic insert
    - Per CONTEXT.md: "Action items include: title, assignee, optional due date. No full action item management UI."

    **Action items API (`/api/sessions/[id]/action-items`):**
    - POST: Create action item. Input: `{ title: string, assigneeId: string, dueDate?: string, category?: string }`
    - PATCH: Update action item (title, status). Input: `{ id: string, title?: string, status?: string }`
    - GET: List action items for session. Filter: optional `?category=xxx`
    - Auth: isSeriesParticipant
    - Sets tenantId, createdById from session context
