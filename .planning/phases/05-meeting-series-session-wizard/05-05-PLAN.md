    **Scoring utility (`src/lib/utils/scoring.ts`):**

    ```typescript
    export function normalizeAnswer(answerType: string, value: number): number
    ```
    - rating_1_5: return value (already 1-5 scale)
    - rating_1_10: return ((value - 1) / 9) * 4 + 1 (maps 1-10 to 1-5)
    - yes_no: return value * 4 + 1 (maps 0->1, 1->5)
    - mood: return value (already 1-5 scale)
    - Other types (text, multiple_choice): skip (not scorable)

    ```typescript
    export function computeSessionScore(
      answers: Array<{ answerType: string; answerNumeric: number | null; skipped: boolean }>
    ): number | null
    ```
    - Filter to numeric answers that are not skipped and have a value
    - Normalize each via normalizeAnswer
    - Return average, rounded to 2 decimal places
    - Return null if no numeric answers exist (Pitfall 7 from RESEARCH.md: handle mixed types correctly)

    **Completion API (`POST /api/sessions/[id]/complete`):**
    - Auth check: only the manager on the series can complete a session
    - Verify session status is "in_progress" (can't complete an already completed session)
    - In a single withTenantContext transaction:
      1. Fetch all session_answers for this session with their question's answerType (JOIN to template_question)
      2. Compute session score using computeSessionScore()
      3. Update session: set status="completed", completedAt=now(), sessionScore=computed score, durationMinutes=difference between startedAt and now() in minutes
      4. Fetch the series for this session (to get cadence, preferredDay, etc.)
      5. Compute next session date: computeNextSessionDate(new Date(), series.cadence, series.cadenceCustomDays, series.preferredDay)
      6. Update series: set nextSessionAt to the computed date (MEET-05: next date auto-computed after completion)
      7. Audit log: "session_completed" with sessionScore and sessionNumber
    - Return: { sessionId, score, nextSessionAt }
