       ```typescript
       export const postSessionPipeline = inngest.createFunction(
         { id: "post-session-ai-pipeline", retries: 3, concurrency: [{ scope: "fn", limit: 5 }] },
         { event: "session/completed" },
         async ({ event, step }) => { ... }
       );
       ```

       Steps (each is an independent `step.run()` for isolated retry):

       a) **"set-ai-status"**: Update session aiStatus to "generating" using withTenantContext(tenantId, managerId, ...)

       b) **"gather-context"**: Call `gatherSessionContext(event.data)` from src/lib/ai/context.ts. This fetches all session data, previous sessions, decrypts manager private notes.

       c) **"generate-summary"**: Call `generateSummary(context)` from AI service. Returns typed AISummary.

       d) **"generate-addendum"**: Call `generateManagerAddendum(context)` from AI service. Returns typed AIManagerAddendum.

       e) **"store-summary"**: Update session row with aiSummary and aiManagerAddendum JSONB columns using withTenantContext.

       f) **"generate-suggestions"**: Call `generateActionSuggestions(context, summary)` from AI service. Returns typed AIActionSuggestions.

       g) **"store-suggestions"**: Update session row with aiSuggestions JSONB column.

       h) **"generate-base-nudges"**: Call `generateNudges(context)` from AI service. Returns typed AINudges. Insert into ai_nudge table with seriesId, tenantId, and targetSessionAt (from series.nextSessionAt if available).

       i) **"finalize"**: Update session aiStatus to "completed", set aiCompletedAt. Log audit event "ai_pipeline_completed".

       Error handling: If any step fails after retries, a final catch sets aiStatus to "failed" on the session. Use `onFailure` callback on the function to handle this.

    2. Update src/inngest/functions/index.ts to export the postSessionPipeline:
