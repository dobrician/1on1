<task type="auto">
  <name>Task 2: Snapshot compute logic, Inngest functions, and query layer</name>
  <files>src/lib/analytics/compute.ts, src/lib/analytics/queries.ts, src/inngest/functions/analytics-snapshot.ts, src/inngest/functions/index.ts, src/inngest/client.ts</files>
  <action>
1. Create `src/lib/analytics/compute.ts`:
   - `computeSessionSnapshot(tx, sessionId, tenantId, reportId, seriesId)`:
     - Fetch session with score, completedAt
     - Fetch session answers joined with template questions (category, answerType, answerNumeric)
     - Compute per-category averages (wellbeing, engagement, performance, career, feedback, mood) -- normalize rating_1_10 to 1-5 scale: `(value - 1) / 9 * 4 + 1`
     - Determine period boundaries (month: first/last of completion month)
     - Upsert rows into analytics_snapshot using `onConflictDoUpdate` on the unique index
     - IMPORTANT: The unique index has nullable columns (userId, teamId, seriesId). PostgreSQL treats NULLs as distinct. For user-level snapshots where teamId is NULL and seriesId might be set, use a WHERE clause approach: delete existing rows matching the key columns (using IS NULL for nulls) then insert fresh rows. This avoids the NULL uniqueness issue.
     - Mark session as ingested: `UPDATE session SET analytics_ingested_at = now() WHERE id = sessionId`
     - All within the same transaction (atomic)

2. Create `src/lib/analytics/queries.ts`:
   - `getScoreTrend(tx, userId, startDate, endDate)`: Query analytics_snapshot for session_score metric by user, ordered by periodStart. Returns `Array<{ date: string; score: number }>`. Fall back to live query on sessions table if no snapshots exist.
   - `getCategoryAverages(tx, userId, startDate, endDate)`: Query analytics_snapshot for category metrics. Returns `Array<{ category: string; avgScore: number; sampleCount: number }>`. Fall back to live query joining session_answers + template_questions.
   - `getSessionComparison(tx, sessionId1, sessionId2)`: Live query (not snapshot) -- fetch per-category scores for two sessions and compute deltas. Returns `Array<{ category: string; score1: number; score2: number; delta: number }>`.
   - `getTeamAverages(tx, teamId, startDate, endDate)`: Aggregate snapshots across team members. Returns `Array<{ category: string; avgScore: number; memberCount: number }>`. Enforce minimum 3 data points for anonymization.
   - `getTeamHeatmapData(tx, teamId, startDate, endDate, anonymize: boolean)`: Returns `Array<{ userId: string; userName: string; category: string; score: number }>`. When anonymize=true, replace userName with "Member 1", "Member 2", etc.

3. Create `src/inngest/functions/analytics-snapshot.ts`:
   - `computeAnalyticsSnapshot`: Inngest function listening to "session/completed" event with 3 retries. Calls computeSessionSnapshot inside step.run.
   - `analyticsSnapshotSweep`: Inngest cron function (`{ cron: "0 3 * * *" }` -- 3 AM daily) using adminDb. Scans for sessions WHERE status='completed' AND analytics_ingested_at IS NULL AND completed_at IS NOT NULL. For each found, sends "session/completed" event to trigger snapshot computation.

4. Register both functions in `src/inngest/functions/index.ts`.

5. Add to Inngest client Events type if needed (the existing "session/completed" event already has the right shape -- no changes needed if snapshot function listens to same event).
