  <name>Task 2: Create .github/workflows/e2e.yml CI workflow</name>
  <files>.github/workflows/e2e.yml</files>
  <action>
Create `.github/workflows/e2e.yml` with a GitHub Actions job that:

1. Triggers on `push` to `main` and on `pull_request` targeting `main`
2. Uses `ubuntu-latest` (matches existing `migrate.yml` pattern)
3. Sets up Bun via `oven-sh/setup-bun@v2`
4. Installs dependencies with `bun install --frozen-lockfile`
5. Installs Playwright browsers — use `npx playwright install --with-deps chromium` (installs only Chromium + system deps; not all browsers — matches Phase 28 Chromium-only scope)
6. Seeds the test database before running tests — runs `bun run db:seed` with `DATABASE_URL` from secrets
7. Runs `bun run test:e2e -- --project=chromium` (only chromium project; setup project runs automatically as a dependency per playwright.config.ts)
8. Uploads the Playwright HTML report as an artifact on failure (using `if: failure()` + `actions/upload-artifact@v4`)

**Required CI secrets (document in the job env section):**
- `DATABASE_URL` — already used by migrate.yml; reuse the same secret
- `NEXTAUTH_SECRET` — needed for Auth.js to sign session tokens during auth setup
- `AUTH_SECRET` — same as NEXTAUTH_SECRET (Auth.js v5 uses AUTH_SECRET)

**Important config details:**
- `CI: true` env var must be set — playwright.config.ts reads it for `forbidOnly`, `retries: 2`, `workers: 1`
- `PLAYWRIGHT_BROWSERS_PATH: /home/runner/.cache/ms-playwright` — CI runners have browsers at this path after `playwright install`
- The `webServer` in playwright.config.ts starts the dev server on 4301 — the job must NOT manually start the server; Playwright handles it via `webServer.command`
- `reuseExistingServer: true` in playwright.config.ts is fine for CI since no server will be running

The workflow file:

```yaml
name: Playwright E2E Tests

on:
  push:
    branches: [main]
  pull_request:
    branches: [main]

jobs:
  e2e:
    runs-on: ubuntu-latest
    timeout-minutes: 20
    env:
      CI: true
      DATABASE_URL: ${{ secrets.DATABASE_URL }}
      NEXTAUTH_SECRET: ${{ secrets.NEXTAUTH_SECRET }}
      AUTH_SECRET: ${{ secrets.AUTH_SECRET }}
      ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}
      PLAYWRIGHT_BROWSERS_PATH: /home/runner/.cache/ms-playwright

    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Setup Bun
        uses: oven-sh/setup-bun@v2
        with:
          bun-version: latest

      - name: Install dependencies
        run: bun install --frozen-lockfile

      - name: Install Playwright browsers
        run: npx playwright install --with-deps chromium

      - name: Seed test database
        run: bun run db:seed

      - name: Run Playwright E2E tests
        run: bun run test:e2e -- --project=setup --project=chromium

      - name: Upload Playwright report
        uses: actions/upload-artifact@v4
        if: failure()
        with:
          name: playwright-report
          path: playwright-report/
          retention-days: 7
```

Note: `ANTHROPIC_API_KEY` is needed because the corrections spec triggers the AI validation endpoint in the correction form flow. Without it, the AI feedback step may error. Add it from `secrets.ANTHROPIC_API_KEY`.

Write this file exactly as shown. The `bun run test:e2e -- --project=setup --project=chromium` pattern explicitly specifies the setup project first and chromium for content tests, which matches the `chromium` project's `dependencies: ["setup"]` config.
